OLUSTURULAN BRANCH I GÖRMEK

git branch= olusturdugumuz branch lari ve master i gösterir.
= hangi branch aktif ise yaninda * vardir.
BRANCLAR ARASI GECIS YAPMAK (Branch tan Master a veya Master dan Branch a)
git checkout branch_isim= yazip enter a basinca istenilen branch a gecilir.
git log --oneline yapildiginda Head dan sonra hangisi yaziyorsa o branchtayizdir
=>hashcode (HEAD - said, master) = said branch indayizdir.
--------------------------------------------------------------------------------

BRANCH BIRLESTIRME (feature i Master ekleme)

git merge branch_isim= branch lari birlestirir.
  1. Ana Branch hangisi ise o branch a gecilmeli (git checkout branch_isim).
  2. git merge yan_branch_isim yazilarak Ana Branch(genelde master) a yerlestirir.
  3. git log --oneline yapilarak iki branch taki veriyonlarin birlestigi görülür.

BRANCH SILME

git branch -d branch_ismi= yazarsak istenilen branch silinir.

BRANCH ISMI DEGISTIRME

git branch -m branch_ismi= aktif olan branch in ismini degistirir.

MERGE CONFLICT

Birlestirilecek commit lerde ayni dosyanin ayni satirinda birbirinden farkli degisiklikler varsa bu durumdaki catismaya denir.
Ekip calismalarinda karsilasabilir.
Remote ile Local in birlestirilmesinde(git push, git pull) ve Branch birlestirmelerinde (git merge branch_isim) karsilasilir.
git pull ve git push edildiginde karsilasilir, 2 cözüm vardir.
  • ya gelen kodlar duzenlenip git add .n git commit -m "..."" git push gönderilir
  • ya da git merge -abort>ENTER ile vazgecebiliriz(gecici bir hamledir)

Cakisma olmamasi icin 5 adimli tavsiye:

  1. github da(REMOTE) dosayi olustur.
  2. Local e git pull et.
  3. Local de Branch olustur.(feature)
  4. Local Branch daki degisikligi Remote Branch a gönder.
  5. Remote Branch takini Master Branch ile birlestir.
